<task type="auto">
  <name>Task 1: Create bare Laravel 12 project and install Livewire 4</name>
  <files>composer.json, package.json, vite.config.js, resources/views/layouts/app.blade.php, resources/css/app.css</files>
  <action>
    1. Create a new Laravel 12 project in the current directory. Since this repo already has .planning/ and .git, use `composer create-project laravel/laravel:^12.0 temp-laravel && mv temp-laravel/* temp-laravel/.* . 2>/dev/null; rm -rf temp-laravel` to scaffold into the existing repo. When prompted for starter kit, select "None" — do NOT use the Livewire starter kit (it bundles auth scaffolding which is Phase 2 scope). Alternatively, use a non-interactive approach.

    2. Enforce the PHP 8.4 target (locked decision): Update `composer.json` to set `"php": "^8.4"` in the `require` section, replacing the default `"php": "^8.2"` that Laravel 12 ships with. Then run `composer validate` to confirm the constraint is accepted.

    3. Install Livewire 4: `composer require livewire/livewire`

    4. Create the Livewire layout: `php artisan livewire:layout` — this generates `resources/views/layouts/app.blade.php` with @livewireStyles and @livewireScripts directives.

    5. Verify Tailwind CSS v4 is configured by default — `package.json` should have `@tailwindcss/vite` and `vite.config.js` should reference the Tailwind plugin. If not present, add them:
       - `npm install -D @tailwindcss/vite`
       - Add `import tailwindcss from '@tailwindcss/vite'` and `tailwindcss()` to the plugins array in `vite.config.js`

    6. Ensure `resources/css/app.css` imports Tailwind via `@import "tailwindcss"` (Tailwind v4 CSS-first config). Do NOT create a `tailwind.config.js` — Tailwind v4 uses CSS-first configuration.

    7. Run `npm install && npm run build` to verify asset compilation works.

    8. Run `php artisan serve` briefly to verify the app starts without errors (test with curl or just check exit code).

    IMPORTANT: Do NOT use the Livewire starter kit. It installs Flux UI, Volt, and auth pages (Phase 2 scope). Use bare install + manual Livewire.
  </action>
  <verify>
    <automated>php artisan route:list --json 2>/dev/null && npm run build 2>&1 | tail -5 && php artisan serve &amp; sleep 2 && curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8000 && kill %1</automated>
  </verify>
  <done>Laravel 12 app serves the default route with 200 status. composer.json requires "php": "^8.4". Livewire 4 is in composer.json. npm run build succeeds with Tailwind v4 assets compiled. Layout file exists with Livewire directives.</done>
</task>

<task type="auto">
  <name>Task 2: Configure development tooling and Blizzard API credential wiring</name>
  <files>pint.json, config/services.php, .env, .env.example, app/Actions/.gitkeep, app/Services/.gitkeep</files>
  <action>
    1. Create `pint.json` in project root with strict_types enforcement:
       ```json
       {
           "preset": "laravel",
           "rules": {
               "declare_strict_types": true
           }
       }
       ```

    2. Add Blizzard credentials to `.env` (with placeholder values):
       ```
       BLIZZARD_CLIENT_ID=your-client-id
       BLIZZARD_CLIENT_SECRET=your-client-secret
       BLIZZARD_REGION=us
       ```

    3. Add the same keys (with empty/default values) to `.env.example`:
       ```
       BLIZZARD_CLIENT_ID=
       BLIZZARD_CLIENT_SECRET=
       BLIZZARD_REGION=us
       ```

    4. Add Blizzard service config to `config/services.php` inside the return array:
       ```php
       'blizzard' => [
           'client_id'     => env('BLIZZARD_CLIENT_ID'),
           'client_secret' => env('BLIZZARD_CLIENT_SECRET'),
           'region'        => env('BLIZZARD_REGION', 'us'),
       ],
       ```

    5. Create project convention directories:
       - `app/Actions/.gitkeep` (single-purpose action classes for future phases)
       - `app/Services/.gitkeep` (stateful service classes for future phases)

    6. Verify Pest is available (ships with Laravel 12): `php artisan test` or `./vendor/bin/pest` should run with 0 failures.

    7. Run `./vendor/bin/pint --test` to verify Pint configuration loads correctly.
  </action>
  <verify>
    <automated>php artisan tinker --execute="echo config('services.blizzard.client_id')" 2>/dev/null && ./vendor/bin/pint --test 2>&1 | tail -3 && ./vendor/bin/pest 2>&1 | tail -5</automated>
  </verify>
  <done>config('services.blizzard.client_id') returns the .env value. .env.example has all BLIZZARD_* keys. pint.json exists with declare_strict_types rule. Pest runs with 0 failures. app/Actions/ and app/Services/ directories exist.</done>
</task>
