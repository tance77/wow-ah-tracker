       - `php artisan make:migration create_watched_items_table`

    2. Implement `watched_items` migration (must run AFTER the default users migration):
       ```php
       <?php
       declare(strict_types=1);
       // ...
       Schema::create('watched_items', function (Blueprint $table) {
           $table->id();
           $table->foreignId('user_id')->nullable()->constrained()->nullOnDelete();
           $table->unsignedBigInteger('blizzard_item_id');
           $table->string('name');
           $table->unsignedInteger('buy_threshold');   // integer percentage (e.g., 15 = 15%)
           $table->unsignedInteger('sell_threshold');  // integer percentage (e.g., 15 = 15%)
           $table->timestamps();
       });
       ```
       - `user_id` is nullable FK to users — avoids migration change when Phase 2 adds auth (per user decision)
       - Hard delete only — NO SoftDeletes trait (per user decision)
       - Bare minimum columns only — no icon, category, quality columns (per user decision)

    3. Implement `price_snapshots` migration:
       ```php
       <?php
       declare(strict_types=1);
       // ...
       Schema::create('price_snapshots', function (Blueprint $table) {
           $table->id();
           $table->foreignId('watched_item_id')->constrained()->cascadeOnDelete();
           $table->unsignedBigInteger('min_price');     // copper, NOT gold/float
           $table->unsignedBigInteger('avg_price');     // copper, NOT gold/float
           $table->unsignedBigInteger('median_price');  // copper, NOT gold/float
           $table->unsignedBigInteger('total_volume');  // unsigned big integer
           $table->timestamp('polled_at');              // datetime, NOT Unix integer
           $table->timestamps();

           // CRITICAL: Composite index — cannot be efficiently added after data accumulates in SQLite
           $table->index(['watched_item_id', 'polled_at']);
       });
       ```

    CRITICAL CONSTRAINTS (per user decisions and roadmap):
    - All four price/volume columns MUST be `unsignedBigInteger` — NOT integer, float, or decimal
    - `polled_at` MUST be `timestamp()` — NOT unsignedInteger for Unix time
    - Composite index on `(watched_item_id, polled_at)` MUST be in the create callback
    - `declare(strict_types=1)` on both migration files
    - `watched_items` migration must have a `down()` method with `Schema::dropIfExists('watched_items')`
    - `price_snapshots` migration must have a `down()` method with `Schema::dropIfExists('price_snapshots')`

    4. Run `php artisan migrate` to verify both migrations execute successfully.

    1. Create WatchedItem model (`php artisan make:model WatchedItem -f` or create manually):
       ```php
       <?php
       declare(strict_types=1);
       namespace App\Models;

       use Illuminate\Database\Eloquent\Factories\HasFactory;
       use Illuminate\Database\Eloquent\Model;
       use Illuminate\Database\Eloquent\Relations\BelongsTo;
       use Illuminate\Database\Eloquent\Relations\HasMany;

       class WatchedItem extends Model
       {
           use HasFactory;

           protected $fillable = [
               'user_id',
               'blizzard_item_id',
               'name',
               'buy_threshold',
               'sell_threshold',
           ];

           protected $casts = [
               'blizzard_item_id' => 'integer',
               'buy_threshold' => 'integer',
               'sell_threshold' => 'integer',
           ];

           public function user(): BelongsTo
           {
               return $this->belongsTo(User::class);
           }

           public function priceSnapshots(): HasMany
           {
               return $this->hasMany(PriceSnapshot::class);
           }
       }
       ```
       - NO SoftDeletes (per user decision — hard delete only)
       - Full type hints on all methods (per project convention)
