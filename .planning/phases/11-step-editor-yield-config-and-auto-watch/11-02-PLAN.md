    1. `selectInputItem(int $blizzardItemId, string $name)` — sets selectedInputItemId, selectedInputName, clears inputSearch
    2. `selectOutputItem(int $blizzardItemId, string $name)` — sets selectedOutputItemId, selectedOutputName, clears outputSearch

    4. `addStep(int $inputBlizzardItemId, int $outputBlizzardItemId, int $inputQty, int $outputQtyMin, int $outputQtyMax)`:
       - Validate: inputQty >= 1, outputQtyMin >= 1, outputQtyMax >= outputQtyMin
       - Create ShuffleStep with sort_order = count of existing steps
       - Call autoWatch for both input and output blizzard_item_ids
       - Reset add-step form, set addingStep=false
       - `unset($this->steps)` to refresh computed

    5. `saveStep(int $stepId, int $inputQty, int $outputQtyMin, int $outputQtyMax)`:
       - Find step via `$this->shuffle->steps()->findOrFail($stepId)`
       - Validate same as addStep
       - Update step with new values
       - `unset($this->steps)` to refresh computed

    6. `deleteStep(int $stepId)`:
       - Find step via `$this->shuffle->steps()->findOrFail($stepId)`
       - Delete step (ShuffleStep::boot() deleting event handles orphan cleanup)
       - Renumber remaining steps: `$this->shuffle->steps()->get()->each(fn ($s, $i) => $s->update(['sort_order' => $i]))`
       - `unset($this->steps)` to refresh computed

    7. `moveStepUp(int $stepId)`:
       - Get all steps ordered by sort_order
       - Find index of target step
       - If index < 1, return (already first)
       - Swap sort_order values with previous step, save both
       - `unset($this->steps)` to refresh computed

    8. `moveStepDown(int $stepId)`:
       - Same as moveStepUp but swap with next step
       - If index >= count-1, return (already last)

    9. `private function autoWatch(int $blizzardItemId): void`:
       - Look up CatalogItem by blizzard_item_id to get name (fallback to "Item #ID")
       - `auth()->user()->watchedItems()->firstOrCreate(['blizzard_item_id' => $blizzardItemId], ['name' => $name, 'buy_threshold' => null, 'sell_threshold' => null, 'created_by_shuffle_id' => $this->shuffle->id])`
       - Silent — no toast or notification per CONTEXT.md

    **Validation approach:** Use inline validation with `$this->addError()` for yield fields. Show errors next to fields. Prevent save if invalid. Clear errors on valid save.

    - Keep the `addStep` method signature matching what Plan 01 tests expect
    - The `addStep` method should be callable from the Blade template via `$wire.addStep(...)` passing the selected item IDs and qty values
    - For the "add step" form, the public properties hold the form state, and `addStep()` reads from them OR accepts parameters directly (the Livewire test calls it with parameters, the UI calls it reading properties — support both by having the UI button call a wrapper that passes the property values)
