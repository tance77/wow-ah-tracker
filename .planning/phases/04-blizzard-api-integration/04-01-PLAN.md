Implement BlizzardTokenService that fetches and caches a Blizzard OAuth2 access token using client credentials flow, and register it as a singleton in the service container.

Purpose: Phase 5's ingestion job needs a valid Blizzard API token on every run. This service encapsulates token acquisition, caching (23-hour TTL), and credential handling so downstream consumers never deal with OAuth2 directly.
Output: `BlizzardTokenService` class with `getToken(): string` method; singleton registration in `AppServiceProvider`.
</objective>

<task type="auto">
  <name>Task 1: Create BlizzardTokenService with OAuth2 client credentials and Cache::remember()</name>
  <files>app/Services/BlizzardTokenService.php</files>
  <action>
Create `app/Services/BlizzardTokenService.php` with `declare(strict_types=1)` and namespace `App\Services`.

Class `BlizzardTokenService` with a single public method `getToken(): string`:

1. Use `Cache::remember('blizzard_token', 82800, fn () => ...)` (82800 = 23 hours in seconds).
2. Inside the closure:
   - Log an info message: `'BlizzardTokenService: fetching new access token'`
   - POST to `https://oauth.battle.net/token` using `Http::withBasicAuth(config('services.blizzard.client_id'), config('services.blizzard.client_secret'))->asForm()->timeout(30)->post(...)` with body `['grant_type' => 'client_credentials']`.
   - If response is not successful, throw `RuntimeException` with message including the HTTP status code: `"Blizzard token request failed: HTTP {status}"`.
   - Extract `access_token` from JSON response: `$response->json('access_token')`.
   - If token is empty/null, throw `RuntimeException`: `"Blizzard token response missing access_token"`.
   - Log info: `'BlizzardTokenService: new token acquired'`.
   - Return the token string.

3. Add `->retry(2, 1000)` to the HTTP call chain (after `asForm()`, before `timeout(30)`) for resilience against transient failures. This is Claude's discretion per CONTEXT.md.

Do NOT query the database or validate credentials at boot time (lazy validation per CONTEXT.md decision).
Do NOT use raw Guzzle — must use Laravel Http facade for `Http::fake()` testability.
  </action>

    <automated>php artisan tinker --execute="app(\App\Services\BlizzardTokenService::class); echo 'Resolves OK';"</automated>
  </verify>
  <done>BlizzardTokenService class exists with getToken() method that uses Cache::remember() at 23-hour TTL, Http::withBasicAuth() for token POST, and throws RuntimeException on failure.</done>
</task>
