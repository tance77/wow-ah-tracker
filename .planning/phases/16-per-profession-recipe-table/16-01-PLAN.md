  <action>
Update the PHP section of `crafting-detail.blade.php` to add a `#[Computed]` property that builds the full recipe dataset. Keep the existing mount() and Profession model binding. Add `use FormatsAuctionData;` trait.

The `recipeData()` computed property should:

1. **Eager load** the profession's recipes with the established pattern:
   ```php
   $this->profession->load([
       'recipes.reagents.catalogItem.priceSnapshots' => fn ($q) => $q->latest('polled_at')->limit(1),
       'recipes.craftedItemSilver.priceSnapshots' => fn ($q) => $q->latest('polled_at')->limit(1),
       'recipes.craftedItemGold.priceSnapshots' => fn ($q) => $q->latest('polled_at')->limit(1),
   ]);
   ```

2. **Compute profit** for each recipe using `new RecipeProfitAction()`.

3. **Track staleness** by finding the oldest `polled_at` across all reagent and crafted item price snapshots. Set `stale = true` if oldest is > 60 minutes ago.

4. **Build reagent breakdown** per recipe: array of `{name, quantity, unit_price, subtotal}` for each reagent.

5. **Return structure:**
   ```php
   return [
       'recipes' => [
           [
               'id' => int,
               'name' => string,
               'is_commodity' => bool,
               'reagent_cost' => int|null,
               'profit_silver' => int|null,
               'profit_gold' => int|null,
               'median_profit' => int|null,
               'has_missing_prices' => bool,
               'reagents' => [
                   ['name' => string, 'quantity' => int, 'unit_price' => int|null, 'subtotal' => int|null],
               ],
           ],
       ],
       'stale' => bool,
       'stale_minutes' => int|null,
   ];
   ```

For the Blade template, add a minimal placeholder that embeds the data via `@js($this->recipeData)` so tests can verify the data pipeline. The full UI comes in Plan 02. Example minimal template:

```html
<div x-data="{ ...@js($this->recipeData) }">
    <template x-for="recipe in recipes" :key="recipe.id">
        <div x-text="recipe.name"></div>
    </template>
</div>
```

Add these imports to the PHP section:
- `use App\Actions\RecipeProfitAction;`
- `use App\Concerns\FormatsAuctionData;`
- `use Livewire\Attributes\Computed;`

Follow the exact pattern from 16-RESEARCH.md Pattern 1 (Server-Side Data Preparation). Avoid N+1 queries (Pitfall 1 from research).
