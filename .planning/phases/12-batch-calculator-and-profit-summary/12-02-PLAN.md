  <name>Task 1: Build Alpine.js batch calculator UI</name>
  <files>resources/views/livewire/pages/shuffle-detail.blade.php, tests/Feature/ShuffleBatchCalculatorTest.php</files>
  <action>
    Add the batch calculator Blade/Alpine section to `shuffle-detail.blade.php` below the step editor section. Use `wire:ignore` on the calculator container to prevent Livewire DOM morphing from resetting Alpine state (batchQty).

    **Calculator structure (all within an `@if($this->steps->isNotEmpty())` guard):**

    1. **Staleness warning banner** (amber, top of calculator section):
       - `x-show="staleItems.length > 0"`
       - Lists item names with "Xm ago" age labels
       - Amber text on amber-900/20 background, consistent with WoW dark theme

    2. **Input quantity field:**
       - `<input type="number" min="1" x-model.number="batchQty">` with default value 1
       - Label: "Input Quantity"
       - WoW dark theme styling (bg-wow-darker, border-gray-600, focus:border-wow-gold)

    3. **Step breakdown table (compact):**
       - One row per step from `cascade` getter
       - Columns: Step # | Input (icon + name + qty) | Output (icon + name + qty min-max) | Yield ratio
       - Item icons as 20x20 `<img>` tags alongside names
       - Show min and max output quantities when they differ: "12 - 36"
       - Use `x-for="(row, i) in cascade"` loop

    4. **Profit summary section (five rows):**
       - Total Cost: `formatGold(totalCostMin)` / `formatGold(totalCostMax)` — neutral gray text
       - Gross Value: `formatGold(grossValueMin)` / `formatGold(grossValueMax)` — neutral gray text
       - AH Cut (5%): computed as `Math.round(grossValue * 0.05)` — muted gray text
       - Net Profit: green when >= 0, red when < 0 (per user decision)
       - Break-even price: `formatGold(breakEven)` in wow-gold color — only Min column (Max shows "---")
       - When `canCalculate` is false: show "Cannot calculate -- missing prices for: [item names]" message

    5. **Missing price handling:**
       - Items with no snapshots show "--" for monetary values
       - Yield quantities still cascade and display (only monetary values suppressed)

    **Alpine component (`batchCalculator` function):**

    Define as an inline function in the Blade template (not a separate JS file — consistent with step editor Alpine pattern). Include:
    - `batchQty: 1` reactive property
    - `get cascade()` — iterates steps, cascades min/max yields: `outMin = Math.floor(qtyMin * step.output_qty_min / step.input_qty)`
    - `get staleItems()` — filters unique item IDs where `prices[id]?.stale === true`
    - `get canCalculate()` — checks first input price and last output price are non-null
    - `get totalCostMin/Max()` — first step input qty * first input price
    - `get grossValueMin/Max()` — last step output qty * last output price
    - `get netProfitMin/Max()` — `Math.round(grossValue * 0.95) - totalCost`
    - `get breakEven()` — `Math.floor(Math.round(grossValueMin * 0.95) / batchQty)` — per-input-unit max cost
    - `formatGold(copper)` — JS mirror of PHP formatGold: `g = Math.floor(copper / 10000)`, `s = Math.floor((copper % 10000) / 100)`, `c = copper % 100`, with zero-suppression and negative support

    **Cost/Value model (per user decisions):**
    - Cost = only first step's input (what we buy)
    - Value = only final step's output (what we sell on AH) x 0.95
    - Intermediate items have no cost (they are produced, not bought)

    **Styling per user decisions:**
    - WoW dark theme: bg-wow-dark card, bg-wow-darker inputs, text-wow-gold accents
    - Data-dense and scannable, like TSM/Auctioneer addon aesthetics
    - Compact table with minimal padding
    - Net Profit: text-green-400 when >= 0, text-red-400 when < 0

    **Tests to add:**
    - Calculator section renders "Batch Calculator" heading when shuffle has steps
    - Calculator section does NOT render when shuffle has no steps
    - Calculator section contains `x-data="batchCalculator"` (verifies Alpine binding)
    - priceData JSON is embedded in the page (spot-check for a known blizzard_item_id)
