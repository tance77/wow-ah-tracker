Replace the minimal placeholder Blade template from Plan 01 with the full interactive table. Keep the PHP section (imports, mount, recipeData computed) unchanged. Only modify the Blade HTML below the `?>` marker.

**Alpine.js x-data component** (following Pattern 2 from research):

Initialize with:
```javascript
{
    ...@js($this->recipeData),
    sortBy: 'median_profit',
    sortDir: 'desc',
    searchQuery: '',
    expandedRow: null,
    formatGold(copper) { /* mirror PHP formatGold — see research code example */ },
    get sortedRecipes() { /* partition + sort — see research */ },
    toggleSort(col) { /* toggle asc/desc — see research */ },
    toggleExpand(id) { /* accordion — only one open */ }
}
```

**formatGold(copper):** Port from `FormatsAuctionData.php` to JavaScript. Must handle null (return em dash), zero, negative values, copper-only amounts. See research Code Examples section for exact implementation.

**sortedRecipes getter:** Partition recipes into `normal` (is_commodity && !has_missing_prices) and `bottom` (everything else). Sort each partition independently by current sortBy/sortDir. Filter both by searchQuery first. Concatenate normal + bottom. This ensures missing-price and non-commodity rows always sort to bottom per user decision.

**Page structure (top to bottom):**

1. **Header slot** — Profession name + recipe count (e.g., "Alchemy" with "47 recipes" subtext). Include a back link to `/crafting`.

2. **Staleness banner** — Conditionally shown with `x-show="stale"` and `x-cloak`. Amber styling matching existing pattern: `border-amber-700/50 bg-amber-900/20`. Text: "Price data may be stale -- last updated X minutes ago". Use warning SVG icon. No dismiss button (per research recommendation — always visible while stale).

3. **Search filter** — Text input above the table. Placeholder "Search recipes...". Styled with WoW dark theme input styling. Use `x-model="searchQuery"` for reactive filtering.

4. **Recipe count indicator** — Show filtered count, e.g., "Showing X of Y recipes".

5. **Table** — Wrapped in `overflow-x-auto` for mobile horizontal scroll (per research recommendation).

   **Table headers** (all clickable for sorting):
   - Recipe Name (left-aligned, sorts alphabetically)
   - Reagent Cost (right-aligned)
   - T1 Profit (right-aligned) — label "Tier 1"
   - T2 Profit (right-aligned) — label "Tier 2"
   - Median Profit (right-aligned)

   Each header: `@click="toggleSort('column_key')"` with cursor-pointer. Show sort direction indicator (chevron SVG, rotated for asc/desc) only on the active sort column.

   **Table rows** — Use `<template x-for="recipe in sortedRecipes" :key="recipe.id">` wrapping a `<tbody>` per recipe (to support the expansion row).

   Each row:
   - Click anywhere on the row triggers `toggleExpand(recipe.id)` — cursor-pointer
   - **Name cell:** Chevron icon (rotated 90deg when expanded via `:class="{ 'rotate-90': expandedRow === recipe.id }"`), recipe name, and conditional amber "missing prices" badge (`x-if="recipe.has_missing_prices"`)
   - **Reagent cost cell:** Always shown (even for non-commodity). Use `formatGold(recipe.reagent_cost)` or em dash if null
   - **Profit cells (conditional on is_commodity):**
     - If `recipe.is_commodity`: Show T1, T2, Median profit with `formatGold()`. Color: green if > 0, red if < 0, gray if null (em dash). If has_missing_prices, show em dash for all profit columns.
     - If `!recipe.is_commodity`: Single `<td colspan="3">` with "Realm AH -- not tracked" (italic, gray, centered). Omit the 3 individual profit `<td>` elements when using colspan (Pitfall 5 from research).
   - Non-commodity rows: add `opacity-50` class to the `<tr>` (slightly dimmed per user decision)
   - Row hover: `hover:bg-wow-darker/50` with `transition-colors`
   - Row border: `border-b border-gray-700/40`

   **Expansion row** — Shown via `<template x-if="expandedRow === recipe.id">` directly after the main row within the same `<tbody>`:
   - Background: `bg-wow-darker/30`
   - Colspan 5 (or full width)
   - Padding: `px-8 py-3`
   - Contains a sub-table or styled list of reagents:
     - Each reagent: quantity, name, unit price (formatGold), subtotal (formatGold)
     - Format: "2x Resonance Crystal @ 20g = 40g" style, or a clean mini-table with columns
     - If unit_price is null, show "no price" in gray italic
   - Track expandedRow by recipe ID (not array index) to avoid Pitfall 4

6. **Empty state** — If `sortedRecipes.length === 0` (after search filter), show "No recipes found" message.

**Styling notes:**
- Table text: `text-sm text-gray-300` for data, `text-xs text-gray-400` for headers
- Use the WoW dark theme consistently
- Table: no outer border, subtle row borders only
- Mobile: `overflow-x-auto` wrapper with `min-w-[600px]` on the table itself

**Important anti-patterns to avoid (from research):**
- Do NOT use Livewire wire:click for sort — Alpine only, no server round-trips
- Do NOT use array index for expandedRow — use recipe.id
- Do NOT render 3 profit `<td>` elements AND a colspan `<td>` — use `x-if` to pick one or the other
- Do NOT separate reagent data fetch — it's already in the initial payload
