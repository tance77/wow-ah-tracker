### Phase 1: Project Foundation
- `watched_items.user_id` FK uses `nullOnDelete()` — deleting a User orphans WatchedItem rows with NULL `user_id`. `FetchCommodityPricesJob` calls `WatchedItem::all()` and continues writing snapshots for orphaned items. No display impact (all queries use `auth()->user()->watchedItems()`), but accumulates dead rows. Consider `cascadeOnDelete()` or a scheduled cleanup if user deletion becomes a production scenario.
