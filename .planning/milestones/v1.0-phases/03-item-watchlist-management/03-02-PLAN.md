**PHP block (above `?>`):**
- `declare(strict_types=1);`
- Imports: `App\Models\CatalogItem`, `Illuminate\Support\Collection`, `Livewire\Attributes\Computed`, `Livewire\Attributes\Layout`, `Livewire\Volt\Component`
- Class: `new #[Layout('layouts.app')] class extends Component`
- Public properties:
  - `string $search = ''` — combobox search input
  - `string $manualItemId = ''` — manual Blizzard ID input
- Computed property `watchedItems(): Collection` — returns `auth()->user()->watchedItems()->orderBy('name')->get()`
- Computed property `catalogSuggestions(): array` — returns `[]` when `strlen($this->search) < 2`, otherwise `CatalogItem::where('name', 'like', "%{$this->search}%")->limit(15)->orderBy('name')->get(['id', 'name', 'blizzard_item_id'])->toArray()`
- Method `addFromCatalog(int $catalogId): void`:
  - `$catalog = CatalogItem::findOrFail($catalogId);`
  - `auth()->user()->watchedItems()->firstOrCreate(['blizzard_item_id' => $catalog->blizzard_item_id], ['name' => $catalog->name, 'buy_threshold' => 10, 'sell_threshold' => 10])`
  - Reset `$this->search = '';`
- Method `addManual(): void`:
  - Validate: `$this->validate(['manualItemId' => 'required|integer|min:1']);`
  - `$id = (int) $this->manualItemId;`
  - `auth()->user()->watchedItems()->firstOrCreate(['blizzard_item_id' => $id], ['name' => "Item #{$id}", 'buy_threshold' => 10, 'sell_threshold' => 10])`
  - Reset `$this->manualItemId = '';`
- Method `removeItem(int $id): void`:
  - `auth()->user()->watchedItems()->findOrFail($id)->delete();`
- Method `updateThreshold(int $id, string $field, int $value): void`:
  - Guard: `if (! in_array($field, ['buy_threshold', 'sell_threshold'], true)) { return; }`
  - `$item = auth()->user()->watchedItems()->findOrFail($id);`
  - `$item->update([$field => max(1, min(100, $value))]);`

CRITICAL: EVERY database query MUST go through `auth()->user()->watchedItems()` — NEVER use `WatchedItem::query()` or `WatchedItem::find()` directly. This enforces ITEM-05 user isolation.

- Adjacent to combobox (or below), a small form with number input + "Add by ID" button

- Button: `wire:click="addManual"`, styled as secondary action

**Watchlist table:**
- Columns: Item Name, Buy Threshold (%), Sell Threshold (%), Remove
- Dense, scannable rows per CONTEXT.md
- Each `<tr>` has `wire:key="item-{{ $item->id }}"`
- Item name cell: name as primary text, Blizzard ID as `<span class="text-xs text-gray-500 block">ID: {{ $item->blizzard_item_id }}</span>` (secondary text, not a column)
- Threshold cells use inline editing pattern:
  - `x-data="{ editing: false, saved: false }"`
  - Display mode: `<span x-show="!editing" @click="editing = true" class="cursor-pointer hover:text-wow-gold">{{ $item->buy_threshold }}%</span>`
  - Edit mode: `<input type="number" min="1" max="100" x-show="editing">`
  - Use `wire:change="updateThreshold({{ $item->id }}, 'buy_threshold', $event.target.value)"` (NOT wire:model on computed collection)
  - On blur: `@blur="editing = false; saved = true; setTimeout(() => saved = false, 1000)"`
  - Auto-focus when entering edit: `x-ref="buyInput"` + `x-init="$watch('editing', v => v && $nextTick(() => $refs.buyInput.focus()))"`
  - Save feedback: `<span x-show="saved" x-transition class="text-green-400 text-xs ml-1">Saved</span>`
- Remove button: `wire:click="removeItem({{ $item->id }})"`, red text, no confirmation modal per CONTEXT.md
- Table styling: `border-gray-700/50` borders, `bg-wow-dark` background, `text-gray-200` text
