    **PriceFetchAction changes** (`app/Actions/PriceFetchAction.php`):

    1. Change the `@return` PHPDoc to:
       ```php
       /** @return array{listings: array<int, array<string, mixed>>, lastModified: ?string, rawBody: string} */
       ```
    2. After the `$response->successful()` check and before the filtering logic, capture:
       ```php
       $lastModified = $response->header('Last-Modified') ?: null;
       $rawBody = $response->body();
       ```
       IMPORTANT: `$response->header()` returns `''` (empty string) when absent — normalize to `null` with `?: null`.

    3. Compute the hash from the raw body BEFORE filtering:
       ```php
       $hash = md5($rawBody);
       ```
       Use MD5 (not SHA256) — sufficient for dedup, faster, and this is not a security use case.

    4. Keep the existing `$auctions = $response->json('auctions', [])` and filter logic unchanged.

    5. Change the return statement to:
       ```php
       return [
           'listings' => array_values(array_filter(
               $auctions,
               fn (array $entry): bool => in_array($entry['item']['id'], $itemIds, strict: true)
           )),
           'lastModified' => $lastModified,
           'rawBody' => $rawBody,
       ];
       ```

    6. Update the Log::info format string to reference `$auctions` count (not the return value, since return shape changed).

    **FetchCommodityPricesJob changes** (`app/Jobs/FetchCommodityPricesJob.php`):

    1. Add import: `use App\Models\IngestionMetadata;`

    2. Wrap the fetch call in a try/catch for `\RuntimeException`:
       ```php
       try {
           $result = ($fetchAction)($itemIds);
       } catch (\RuntimeException $e) {
           Log::error('FetchCommodityPricesJob: fetch failed, skipping cycle', [
               'error' => $e->getMessage(),
           ]);
           $meta = IngestionMetadata::singleton();
           $meta->increment('consecutive_failures');
           return;
       }
       ```

    3. After successful fetch, load metadata and apply dedup gate:
       ```php
       $meta = IngestionMetadata::singleton();

       // Primary gate: Last-Modified header comparison
       if ($result['lastModified'] !== null && $result['lastModified'] === $meta->last_modified_at) {
           Log::info('FetchCommodityPricesJob: data unchanged (Last-Modified match), skipping write');
           return;
       }

       // Fallback gate: response body hash (when Last-Modified absent)
       $hash = md5($result['rawBody']);
       if ($result['lastModified'] === null && $hash === $meta->response_hash) {
           Log::info('FetchCommodityPricesJob: data unchanged (hash match), skipping write');
           return;
       }
       ```

    4. Update the `$listings` variable reference to use `$result['listings']`:
       ```php
       $listings = $result['listings'];
       ```
       The rest of the grouping/aggregate/write loop stays the same.

    5. After the write loop (after the existing `foreach ($watchedItems as $watchedItem)` block), update metadata:
       ```php
       $meta->update([
           'last_modified_at'     => $result['lastModified'],
           'response_hash'        => $hash,
           'last_fetched_at'      => now(),
           'consecutive_failures' => 0,
       ]);
       ```

    6. Keep the existing "snapshots written" log line.

    **Key decisions per CONTEXT.md (locked):**
    - Dedup gate applies globally (entire API response), not per-item
    - When dedup gate blocks, log at info level
    - On API failure: catch exception, log error, increment consecutive_failures, skip cycle
    - No Laravel job retries — 15-minute scheduler is the natural retry
    - Reset consecutive_failures to 0 on successful fetch
    - Hash the FULL raw response body (before filtering) — not filtered listings
