    Create `tests/Feature/DataIngestion/PriceAggregateActionTest.php`:

    1. `declare(strict_types=1)` at top
    2. `use App\Actions\PriceAggregateAction;`

    **Tests to write (each as a Pest `it()` block):**

    **Test 1: empty listings return all zeros**
    ```php
    it('returns zero metrics for empty listings', function (): void {
        $action = new PriceAggregateAction;
        $result = $action([]);
        expect($result)->toBe([
            'min_price' => 0, 'avg_price' => 0, 'median_price' => 0, 'total_volume' => 0,
        ]);
    });
    ```

    **Test 2: single listing returns its values**
    - Input: `[['unit_price' => 150000, 'quantity' => 200]]`
    - Expected: min=150000, avg=150000, median=150000, total_volume=200

    **Test 3: multiple listings compute correct min, avg, total_volume**
    - Input: the 224025 fixture data: [{150000, 200}, {175000, 50}]
    - min=150000, total_volume=250
    - avg = round((150000*200 + 175000*50) / 250) = round(38750000/250) = 155000
    - median: sorted same order, cumulative: 200 at 150000 (>=ceil(250/2)=125) -> median=150000

    **Test 4: frequency-distribution median — large quantity bucket dominates**
    This is THE critical test. A naive sort would give the wrong answer.
    - Input: `[['unit_price' => 100000, 'quantity' => 500], ['unit_price' => 200000, 'quantity' => 10], ['unit_price' => 300000, 'quantity' => 5]]`
    - Total volume: 515, medianPosition: ceil(515/2) = 258
    - Cumulative at 100000: 500 (>= 258) -> median = 100000
    - Naive sort of 3 unique prices would pick 200000 (middle) — WRONG
    - `expect($result['median_price'])->toBe(100000);`

    **Test 5: even total volume median**
    - Input: `[['unit_price' => 100000, 'quantity' => 2], ['unit_price' => 200000, 'quantity' => 2]]`
    - Total=4, medianPosition=ceil(4/2)=2, cumulative at 100000=2 (>=2) -> median=100000

    **Test 6: all return values are integers**
    - Use the multiple-listing input from Test 3
    - Assert each value with `->toBeInt()`

    **Test 7: avg_price rounds correctly (not truncates)**
    - Input: listings that produce a non-integer weighted average
    - `[['unit_price' => 100, 'quantity' => 1], ['unit_price' => 200, 'quantity' => 2]]`
    - totalValue = 100 + 400 = 500, totalVolume = 3
    - avg = round(500/3) = round(166.667) = 167 (not 166 from truncation)
    - `expect($result['avg_price'])->toBe(167);`
  </action>
