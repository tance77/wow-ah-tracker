  <action>
Create `app/Actions/PriceFetchAction.php` with `declare(strict_types=1)` and namespace `App\Actions`.

Class `PriceFetchAction` with:

1. **Constructor:** Accept `BlizzardTokenService $tokenService` via constructor injection (promoted readonly property).

2. **`__invoke(array $itemIds): array` method:**
   a. Read region from config: `$region = config('services.blizzard.region', 'us');`
   b. Get token: `$token = $this->tokenService->getToken();`
   c. Fetch commodities:
      ```php
      $response = Http::withToken($token)
          ->retry(2, 1000)
          ->timeout(30)
          ->get("https://{$region}.api.blizzard.com/data/wow/auctions/commodities", [
              'namespace' => "dynamic-{$region}",
          ]);
      ```
   d. If response is not successful, log error with status code and throw `RuntimeException`: `"Blizzard commodities fetch failed: HTTP {status}"`.
   e. Extract auctions array: `$auctions = $response->json('auctions', []);`
   f. Log info with total auction count and watched item count: `"PriceFetchAction: {total} total listings, filtering to {count} watched items"`.
   g. Filter to only watched items using `array_values(array_filter(...))`:
      ```php
      return array_values(array_filter(
          $auctions,
          fn(array $entry): bool => in_array($entry['item']['id'], $itemIds, strict: true)
      ));
      ```
      Use `array_values()` to re-index the filtered array (avoids sparse keys from `array_filter`).

**Critical constraints:**
- Bearer token goes in `Authorization` header via `withToken()` — NEVER as a query parameter.
- `namespace` is a query parameter, NOT part of the URL path.
- The action does NOT query the database — `$itemIds` comes from the caller (Phase 5 job).
- Return raw Blizzard JSON arrays (not Eloquent models) — Phase 5 handles aggregation.
- 30-second timeout is mandatory (CONTEXT.md locked decision) because the payload is ~70K+ listings.
